# 测试用例管理平台 — 通用能力与设计

## 1. 背景与目标

本文档描述一个**通用测试用例管理平台**的能力清单与设计，参考 Jira 等最佳实践，面向小团队（几人～十几人）、以 QA 为主、开发偶尔参与。平台覆盖：

- **用例库**：编写、分类、检索用例
- **计划与执行**：测试计划、测试轮次、执行结果记录与统计
- **缺陷联动**：失败用例关联缺陷（本地缺陷 + 外部单号如 Jira），双向追溯

后续可决定做成独立系统或集成进现有项目（如 device_management）。

---

## 2. 方案选型：缺陷与联动（混合模式）

| 维度       | 方案一：轻量一体化     | 方案二：以 Jira 为中心 | 方案三：混合（采用）           |
|------------|------------------------|------------------------|--------------------------------|
| 缺陷来源   | 仅本系统内建缺陷单     | 仅关联 Jira Issue Key  | 本地缺陷 + 关联外部单号        |
| 优点       | 无外部依赖             | 不重复建缺陷           | 无 Jira 也能用，有 Jira 可对接 |
| 缺点       | 与 Jira 并用需双维护   | 强依赖 Jira            | 模型与 UI 稍复杂               |

**结论**：采用方案三，兼顾无 Jira 场景与后续对接 Jira 的扩展。

---

## 3. 信息模型（核心实体与关系）

### 3.1 项目 / 空间

- **项目（Project）**：顶层容器，对应一个产品或迭代范围。小团队可先支持单项目，模型保留多项目扩展。
- 属性：名称、标识（key）、描述、创建时间。

### 3.2 测试用例（Test Case）

- 归属某项目；可挂到「模块/目录」下（按功能、端等分层）。
- 属性：标题、前置条件、步骤（序号 + 操作 + 预期结果）、优先级、类型（功能/回归/探索等）、状态（草稿/已评审/已废弃等）、可选标签（Tag）。
- 不做自定义字段，用固定字段 + 标签即可。

### 3.3 测试计划与测试轮次

- **测试计划（Test Plan）**：一次发布或迭代的测试集合，包含要执行的用例列表（按模块/标签选或手动多选）。
- **测试轮次（Test Run）**：某次执行的实例；一个计划下可有多次轮次（如 Round1、回归轮）。
- **Run 内用例（Run-Case）**：某条用例在某次 Run 中的执行记录，避免直接改用例本身。
- Run-Case 属性：执行结果（未执行/通过/失败/阻塞/跳过）、执行人、执行时间、备注；可选附件/截图。

### 3.4 缺陷与联动

- **本地缺陷（Local Bug）**：系统内实体，属性：标题、描述、状态（新建/处理中/已关闭等）、创建人、创建时间；可关联到 Run-Case。
- **外部缺陷（External Bug）**：仅存「外部系统 + 单号」（如 Jira + PROJ-123），展示时用配置的链接模板打开。
- **用例–缺陷关系**：Run-Case 标记失败时可关联一个或多个缺陷（本地或外部）；用例/缺陷两侧均可查看「关联的缺陷」或「关联的 Run-Case/用例」。

### 3.5 关系小结

- Project → Test Case（一对多）
- Project → Test Plan（一对多）
- Test Plan → Test Run（一对多）
- Test Run → Run-Case（一对多）
- Run-Case → Test Case（多对一）
- Run-Case ↔ Bug（多对多：本地缺陷实体或外部单号引用）

## 4. 功能模块

### 4.1 用例管理

- **列表**：按项目筛选，按模块/目录、优先级、类型、标签、关键字搜索；分页。
- **详情/编辑**：单用例查看与编辑（标题、前置条件、步骤、优先级、类型、状态、标签）；步骤为可增删改顺序的列表。
- **创建**：项目下新建用例，必填标题。
- **批量操作**：多选后批量改优先级、类型、标签、状态或批量删除（需确认）；可选批量导出（Excel/CSV）。
- **模块/目录**：项目下层级目录（如「登录」「设备管理」），用例挂到目录下。

暂不做：版本对比、评审工作流、自定义字段。

### 4.2 测试计划与执行

- **计划**：项目下创建计划（名称、描述、用例集合）；用例集合按模块/标签/手动多选；创建后可增删用例。
- **轮次（Run）**：计划下创建轮次；生成该计划下所有选中用例的 Run-Case，初始未执行。
- **执行**：执行视图展示 Run-Case 列表；执行人逐条选择结果（通过/失败/阻塞/跳过）、备注、执行时间；失败时可当场关联缺陷或创建本地缺陷。
- **进度与统计**：计划/轮次维度展示总数、已执行数、通过/失败/阻塞/跳过数量及占比；支持按结果筛选。

暂不做：定时任务、自动重试、复杂审批流。

### 4.3 缺陷与联动

- **关联**：Run-Case 上可「关联已有缺陷」或「新建本地缺陷并关联」；支持选本地缺陷或填外部单号（如 Jira key）+ 链接模板。
- **本地缺陷**：创建时必填标题，选填描述、状态；可与 Run-Case 关联。
- **缺陷列表**：展示本地缺陷 + 外部引用（来源 + 单号 + 链接）；按状态、关联用例/Run 筛选；详情中展示「关联了该缺陷的 Run-Case/用例」。
- **双向追溯**：用例/Run-Case 详情展示关联缺陷；缺陷详情展示关联的 Run-Case 与用例。

暂不做：Jira 自动同步、Webhook、缺陷状态回写。

### 4.4 其他

- **仪表盘/首页**：可选汇总各项目最近计划/轮次、通过率、未执行数、失败关联缺陷数等。
- **导出**：用例列表、计划/轮次执行结果导出 Excel/CSV。

## 5. 权限与多租户（小团队简化版）

- **用户与登录**：账号密码登录（可复用现有系统账号）；用户有显示名称、角色。
- **角色**：**管理员**（项目/用户/配置）、**成员**（用例/计划/执行/缺陷的增删改，不可改系统设置）。不做自定义角色、项目级角色。
- **项目可见性**：多项目时当前所有登录用户可见全部项目；后续可加「项目成员」或可见范围。
- **审计**：可选记录关键操作（创建/删除用例、计划、缺陷）的操作人、时间。

暂不做：SSO、LDAP、细粒度权限、多租户隔离。

---

## 6. 与外部系统对接（可选）

- **缺陷单号与链接**：关联时填写外部单号；系统配置「外部系统名称 + 链接模板」（如 `https://jira.company.com/browse/{key}`），展示为可点击链接。
- **数据范围**：仅引用单号与链接，不拉取 Jira 标题/状态；缺陷列表中外部项只显示来源、单号、链接。
- **配置**：系统设置中配置 0～N 个「外部缺陷系统」（名称 + 链接模板）；执行/缺陷关联处选择来源并填单号。

暂不做：OAuth、自动同步、双向状态回写、从 Jira 导入需求/用例。

---

## 7. 设计小结

| 维度     | 结论                                                                 |
|----------|----------------------------------------------------------------------|
| 信息模型 | Project → 用例/计划/Run → Run-Case；Run-Case 关联本地缺陷或外部单号  |
| 功能     | 用例库（目录、步骤、标签）+ 计划与轮次 + 执行与统计 + 缺陷联动 + 双向追溯；可选仪表盘与导出 |
| 权限     | 管理员 / 成员；多项目先全可见                                       |
| 对接     | 仅「外部单号 + 链接模板」配置，无自动同步                           |
